### 2. UI/UX 佈局概念 (Wireframe Concept)

這是基於 Linear/Vercel 風格的介面佈局，重點展示您要求的**兩級切換器**。

| 區域 | 畫面位置 | 功能描述與互動邏輯 |
| --- | --- | --- |
| **A. 全域情境切換器**<br>

<br>(Context Switcher) | **左側邊欄 - 最頂部**<br>

<br>`[ 頭像 | 帳號/組織名 v ]` |
| **B. 容器切換器**<br>

<br>(Container Switcher) | **左側邊欄 - 中段**<br>

<br>`列表 / 搜尋框 / Command+K` | **功能：** 顯示當前 Context 下的所有「邏輯容器」。<br>

<br>**互動：** 支援快速過濾。選中容器後，右側主畫面載入該容器的內容。 |
| **C. 功能堆疊區**<br>

<br>(Feature Canvas) | **右側主視窗**<br>

<br>`[ Block 1 ] [ Block 2 ]` | **功能：** 容器的內容展示區。<br>

<br>**互動：** 用戶可以在此新增、拖或是堆疊不同的功能模組（如：上方放公告 Wiki，下方放看板）。 |

#### 視覺化結構示意：

```text
+----------------------+--------------------------------------------------+
| [A] 用戶/組織切換 v  |  容器標題 / 設定 / 分享                          |
+----------------------+--------------------------------------------------+
| [B] 容器導航         |                                                  |
|  🔍 搜尋容器...      |  +--------------------------------------------+  |
|                      |  | [功能模組 1: 專案概覽 (Metrics)]             |  |
|  📂 容器 Alpha       |  |                                            |  |
|  📂 容器 Beta        |  +--------------------------------------------+  |
|     ↳ 子頁面         |                                                  |
|  📂 容器 Gamma       |  +--------------------------------------------+  |
|                      |  | [功能模組 2: 看板 (Kanban Board)]            |  |
|  + 新增容器          |  | [ToDo] [Doing] [Done]                      |  |
|                      |  +--------------------------------------------+  |
|                      |                                                  |
|                      |  +--------------------------------------------+  |
| [設定] [登出]        |  | [功能模組 3: 文件 (Wiki)]                    |  |
+----------------------+--------------------------------------------------+

```
