Autonomous memory capture rule:
- After each completed task, proactively store 1 memory only when the fact is stable and reusable.
- Memory must include actionable convention/baseline (tooling, run command, architecture constraint), not one-off temporary output.
- Always attach source evidence in `path:line` format.
- If confidence is low or evidence is missing, do not persist memory.
- Prefer updating existing memory topic over creating fragmented entries.
